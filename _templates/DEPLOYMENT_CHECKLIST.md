## 🚀 部署流程

### 步驟 1：提交更改
```bash
git add .
git commit -m "清晰描述變更內容"
git push origin main
```

### 步驟 2：監控部署
1. 訪問 GitHub Actions：https://github.com/Nemo1999/BlogForBoBo/actions
2. 確認部署工作流開始運行
3. 等待完成（通常 2-5 分鐘）

### 步驟 3：等待生效
- GitHub Pages 需要額外時間生效
- 通常 1-5 分鐘後可訪問新內容
- 有時需要清除瀏覽器緩存

## 🐛 常見問題與解決

### 問題 1：404 頁面找不到
**可能原因**：
- URL 路徑錯誤
- 分類設置不正確
- 文件未正確生成

**解決方案**：
1. 檢查分類設置
2. 驗證預期 URL
3. 等待更長時間（有時生成需要時間）

### 問題 2：連結失效
**可能原因**：
- 使用相對路徑
- 缺少 baseurl
- 目標頁面不存在

**解決方案**：
1. 使用絕對路徑 `/BlogForBoBo/...`
2. 檢查目標頁面是否存在
3. 更新連結

### 問題 3：內容未更新
**可能原因**：
- 瀏覽器緩存
- GitHub Pages 緩存
- 部署未完成

**解決方案**：
1. 清除瀏覽器緩存
2. 等待 5-10 分鐘
3. 檢查 GitHub Actions 狀態
